1. Flussonic Settings
1.1 IsServerMode - (0/1, false/true) - режим взаимодействия с Flussonic Watcher.
1.1.1 Если значение true или 1:
Cлужба будет работать в режиме сервера. Flussonic Watcher сам будет присылать события распознавания в службу при их возникновении. Для этого необходимо в разделе «Уведомления о событиях» в поле «Внешний URL уведомлений о событиях» указать адрес службы типа http://ipaddress:ServerPort/flussonic_event, где ipaddress - адрес хоста на котором запущена служба и ServerPort - значение поля ServerPort (см. 1.2)
1.1.2 Если значение false или 0
Служба будет работать в режиме клиента. Служба будет сама запрашивать новые события у Flussonic Watcher с интервалом указанным в поле EventRequestInterval (см. 1.5)

1.2 ServerPort - порт прослушиваемый службой для получения событий от Flussonic Watcher. Необходимо указать при IsServerMode равном true/1.
1.3 WatcherIPAddress - IP адрес Flussonic Watcher. Необходимо указать при IsServerMode равном false/0.
1.4 WatcherPort - порт Flussonic Watcher. Необходимо указать при IsServerMode равном false/0.
1.5 EventRequestInterval - интервал опроса Flussonic Watcher для получения новых событий в милисекундах. Минимальное значение 300. При значениях меньше 300 мс интервал будет равен 300 мс. Необходимо указать при IsServerMode равном false/0.

2. Orion Settings
2.1 Настройки подключения
2.1.1 IPAddress - IP адрес хоста, на котором запущен Модуль интеграции «АРМ Орион Про»
2.1.2 Port - порт, прослушиваемый Модулем интеграции «АРМ Орион Про»

2.2 Авторизация запросов к Модулю интеграции АРМ «Орион Про». Указывать если в конфигурационном файле настроек Модуля интеграции «АРМ Орион Про» значение поля IsNeedAuth равно 1.
2.2.1 ModuleUserName - имя пользователя для авторизации запросов к Модулю интеграции АРМ «Орион Про»
2.2.2 ModulePassword - пароль для авторизации запросов к Модулю интеграции АРМ «Орион Про»

2.3 Для управления шлагбаумом (и др. устройствами системы) необходимо наличие в системе сотрудника с ключем с типом кода «Удаленное управление». Уровень доступа ключа должен позволять управлять дверьми, которыми должна управлять служба. Логин и код ключа необходимо указать в полях EmployeeUserName (см. 2.1.1) и EmployeePassword (см. 2.1.2) соответственно.
2.3.1 EmployeeUserName - логин ключа типа «Удаленное управление»
2.3.2 EmployeePassword - код ключа типа «Удаленное управление»
2.3.3 TokenLifetime - время жизни токена. Токен генерируется с помощью EmployeeUserName и EmployeePassword и используется при запросах к Модулю интеграции «АРМ Орион Про». Указан в конфигурационном файле настроек Модуля интеграции «АРМ Орион Про». (C:\ProgramData\BolidIntegrServ\settings.ini)

2.4 UseCache -(0/1, false/true) - указывает необходимо ли загружать данные, необходимые для работы службы, в кэш.
2.4.1 Если значение true или 1:
Служба загружает все необходимые для работы данные в кэш через Модуль интеграции АРМ «Орион Про». Данные будут обновляться с интервалом указанным в поле EmployeesUpdatingInterval (см. 1.1) и VisitorsUpdatingInterval (см. 1.2).
Плюсы: 
1. Высокая скорость принятия решения о доступе/запрете
2. Нет необходимости четко указывать гос. номер авто на латинице, заглавными буквами и без пробелов.
Минусы:
1. Малая скорость обновления данных в кэше после внесения изменений через АБД (регулируется EmployeesUpdatingInterval и VisitorsUpdatingInterval).   
2.4.1.1 EmployeesUpdatingInterval - период обновления данных, необходимых для проверки доступа для сотрудника в секундах
2.4.1.1 VisitorsUpdatingInterval - период обновления данных, необходимых для проверки доступа для посетителя в секундах

2.4.2 Если значение false или 0:
Служба запрашивает у Модуля интеграции АРМ «Орион Про» все необходимые данные, непосредственно при выполнении проверки разрешен/запрещен ли доступ.
Плюсы: 
1. Высокая скорость обновления данных после внесения изменений через АБД
2. Необходимо четко указывать гос. номер авто на латинице, заглавными буквам и без пробелов.
Минусы:
1. Возможна малая скорость принятия решения о доступе/запрете при большом количестве данных в БД

2.5 AccesspointsSettings - настройки привязки камер к шлагбаумам
2.5.1 AccesspointId - идентификатор шлагбаума (двери). в БД
2.5.2 EnterCamId - идентификатор камеры из Flussonic Watcher, считывающей номера въезжающих автомобилей 
2.5.3 ExitCamId - идентификатор камеры из Flussonic Watcher, считывающей номера выезжающих автомобилей
